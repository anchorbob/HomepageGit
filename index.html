About me

I got my master's degree in Department of Electrical and Computer Engineering in University of Michigan in Dec 2015.

After that, I started my career in Oracle as a CPU Performance Engineer. I ran Java and Scala benchmarks on different platforms, evaluate their performance, analyze and solve performance issues. The performance comparisons went 3 categories:

  • Performance on different platforms (like SPARC VS x86)
  • Performance on Emulation (Next-gen on FPGA VS Last-gem)
  • Performance on different OS (like Linux-SPARC VS Solaris-SPARC)

The issues I analyzed went across 3 levels:

  • Hardware design issues (micro-architect flaws, long latency event analysis)
  • JVM issues and JVM flags tuning
  • OS/kernel/call stack efficiency analysis

I am an enthusiast of programming and movies. Any topics regarding programming, system performance and movies are welcome to discuss with me.

Some Course Projects

Network File Server

In this project, we group implemented a network file server using knowledge of socket programming, remote procedure call, hierarchical file system and secure communication. More specifically, we implemented fs_session() to start a new session for a client, fs_create(),fs_delete() to create/delete file or directory, fs_writedata(),fs_readdata() to write/read data to/from a file. We also used lock() to handle race condition between multiple clients.

MultiThread Programming

In this project, we group implemented a thread library containing lock(),unlock(),wait(),signal(),broadcast(),yield(),join() for multithread programming using. We used lower level library <interrupt>,<guard> to guarantee the atomicity in kernal codes. In addition, we used opaque pointer idiom to add data/functions for a class without changing that class's header file. We also used getcontext(),makecontext(),swapcontext() to manipulate context of threads. Finally, we used our implemented thread library to support an SSTF(Shortest Seek Time First) disk scheduler.

External Pager

In this project, we implemented an external pager for allocating memories. We implemented a fault handler to handle read/write fault and page fault. When page fault happens, we used clock algorithm to evict a virtual page from physical memory.
